# Decisions

Architectural and process decisions for the PM Knowledge Base. Complements `PLAN.md`'s in-phase decisions log: that file captures tactical decisions during a single phase; this file captures decisions that span phases or shape the system long-term.

For tools/capabilities that are simply *out of scope right now* (known answer, just not yet), see `DEFERRED.md`. The "Decisions deliberately deferred" section below is for *open questions* — we don't yet know the answer.

**Maintenance rule:** When a non-trivial choice is made or explicitly deferred, add a dated entry. Don't delete entries — supersede them by adding a new dated entry that references the old one. Each entry should answer: *what*, *why*, *what alternatives we considered*, and (if relevant) *what would make us reopen it*.
