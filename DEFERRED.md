# Deferred Items

Tools, capabilities, and components intentionally out of scope right now but likely needed later. Each entry records *what*, *why we're not doing it now*, and *what would make us revisit*.

This is different from `DECISIONS.md`'s "Decisions deliberately deferred" section: that section tracks open *questions* we haven't decided. Items here are *known answers* we're simply not building yet.

**Maintenance rule:** When a tool or capability surfaces and we punt it, add an entry. When we adopt one, remove it from this file (or briefly note in `DECISIONS.md` if the adoption was a real decision).
